# Botanical Intelligence — 技術説明

## システムアーキテクチャ

100台のエッジデバイス（M5Stack Compute Kit）のそれぞれに、小規模言語モデル（TinySwallow / SakanaAI）をインストールし、インターネットがないローカル環境下で各デバイスが互いにトークンをリレー形式で引き継ぎながら分散型に文章を生成していく「森林型」のSLMネットワークを構築している。

またSLMの演算は言語から変換されたベクトル以外に、周囲の森林の生体データを用いることでエッジデバイスが置かれた周囲の環境から直接データを取得し、その場と呼応しながら出力を生成していくアーキテクチャを構築している。

## 生成プロセス

具体的な生成イメージ:

1. 別のデバイス・あるいは近くに設置されたマイクから受け渡しされた文章（例：天気がいいね）と、近くの植物の生体信号をSLMに入力
2. 短い続きの文章を出力（例：お日さまはXYZ *植物の生体信号からハルシネーションが誘発されて生成文章が崩れる）
3. 出力された文章がまた次のデバイスの入力となり、一つの入力が森林の樹木のように枝分かれしていく

## BI（ボタニカル・インテリジェンス）とは

こうした植物の信号や森林のようなネットワークを模倣したSLM群を、「BI＝ボタニカル・インテリジェンス」と呼称し、昨今の広がる「便利な」AI応用とは違うベクトルで、AIやLLM(SLM)の持つ可能性を人間の想像力の拡張に応用できないかを探る取り組みである。

## ハードウェア構成

- **メインボード:** NPU（LLM推論に特化した計算機）搭載ボード
- **サブ基板:** 音響・照明・電源周りをコントロール
- **推論方式:** クラウドではなくローカル（オフライン）で推論

## センサー

### 日中：アコースティック・エミッション（AE）
植物は日中、葉の小さな穴（気孔）から水を蒸発させて根から水を引き上げる。その水の通り道（木部）で水の動きの強弱が変化すると微細な気泡が弾け、超音波を含む微弱な音響「アコースティック・エミッション（AE）」が出る。タッチアコースティック・センサは茎に触れるだけでAEを拾い、発生数の変化から鑑賞者が見ることができない水分移動や乾き・ストレスの兆しを記録できる。水があまり動かない夜間はAEが減るため、主に日中に植物の活動の変化を静かに読み取り、計測している。

### 夜間：クロロフィル蛍光
植物は、葉に含まれる緑の色素（葉緑素／クロロフィル）で光エネルギーを吸収して、そのエネルギーを光合成に利用している。ただし、光合成で使い切れなかったエネルギーは、『赤いほのかな光』として葉から外部に放たれることになる。これが「クロロフィル蛍光」である。今回使用する「クロロフィル蛍光画像計測技術」は、青色光を植物に照射することで促される微弱なクロロフィル蛍光の発光現象をカメラで精密に計測し、植物に触れることなく光合成機能の状態を評価する。クロロフィル蛍光の発光の強さや時間変化の様子から、植物の元気さやストレスの兆候を読み取ることができる。夜は太陽光などの外乱光が少なくクロロフィル蛍光が捉えやすいので、植物の元気さやストレス状態の変化をリアルタイムに可視化し、植物の声を聴く。
